üõ† Installation Process

Installing a Kojin Studios bot is designed to be intuitive and secure. Here's the basic process:

Prerequisites: Ensure you have Administrator permissions on your Discord server and a valid Kojin Studios account.

Step 1: Choose Your Bot

Browse our Bot Store and select the bot that best fits your community's needs.

Step 2: Deploy to Server

# Example deployment command
kojin deploy --bot=modguard-pro --server=YOUR_SERVER_ID --config=custom.json

Step 3: Configure Settings

Use our intuitive dashboard or configuration files to customize your bot's behavior.

{
  "bot": "modguard-pro",
  "settings": {
    "automod": {
      "enabled": true,
      "sensitivity": "medium",
      "actions": ["warn", "timeout", "ban"]
    },
    "logging": {
      "channel": "#mod-logs",
      "events": ["all"]
    }
  }
}

üîê Authentication

All API requests require authentication using your API key. Keep your keys secure and never expose them in client-side code.

API Key Authentication

# Include your API key in the Authorization header
curl -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     https://api.kojinstudios.com/v1/bots

Security Note: Store your API keys in environment variables or secure configuration files. Never commit them to version control.

SDK Authentication

If you're using our official SDKs, authentication is handled automatically:

// JavaScript/Node.js
const KojinClient = require('@kojin-studios/sdk');

const client = new KojinClient({
  apiKey: process.env.KOJIN_API_KEY,
  environment: 'production'
});

// Authenticate and get bot status
async function getBotStatus() {
  try {
    const response = await client.bots.getStatus('modguard-pro');
    console.log('Bot Status:', response.data);
  } catch (error) {
    console.error('Authentication failed:', error.message);
  }
}

üìä API Endpoints

Our REST API provides programmatic access to all bot functions and server management features.

Base URL

https://api.kojinstudios.com/v1

Common Endpoints

Get Bot Information

GET /bots/{bot_id}
Authorization: Bearer YOUR_API_KEY

Response:
{
  "id": "modguard-pro",
  "name": "ModGuard Pro",
  "status": "active",
  "uptime": "99.98%",
  "servers": 1247,
  "features": ["automod", "logging", "custom-rules"]
}

Update Bot Configuration

PUT /bots/{bot_id}/config
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{
  "automod": {
    "enabled": true,
    "sensitivity": "high"
  }
}

Response:
{
  "success": true,
  "message": "Configuration updated successfully",
  "timestamp": "2025-06-20T12:00:00Z"
}
